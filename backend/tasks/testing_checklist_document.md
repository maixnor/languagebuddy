### Prompt: Create Testing Checklist Document

**Goal:** Create a `TESTING-CHECKLIST.md` document for systematic manual testing before deploying new features.

**Context:** Need a structured approach to ensure new features are thoroughly tested.

**Target File:** `backend/TESTING-CHECKLIST.md` (new file)

**Details:**
- Document a checklist for pre-deployment validation of features.
- Include sections for:
  - **Weakness Integration:** Verify deficiency tracking, practice, and updates.
  - **Digest System:** Confirm digest creation, profile updates, conversation clearing.
  - **Time Awareness:** Check message timestamps, time gap calculations, natural conversation ending.
- Add a section for post-deployment checks (admin dashboard, daily digest, spot checks, health check).

**Expected Impact:** Improved quality assurance and reduced bugs in new feature deployments.
